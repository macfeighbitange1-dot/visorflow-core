🧠 System Architecture: VisorFlow (or Synapse)
1. High-Level Design PhilosophyThe system follows a Brain-Muscle-Cell architectural pattern, designed for high-concurrency, isolated code execution.The Brain (Rails 8.0): The central nervous system. Manages state, persistence (PostgreSQL), and high-level logic. It does not execute untrusted code; it only issues commands.The Muscle (Node.js/Express): The high-speed orchestrator. It sits on the host machine, translates Rails API calls into Docker socket commands, and handles binary stream multiplexing.The Cells (gVisor/Alpine): Isolated runtime environments. Each "cell" is a container wrapped in a Google gVisor (runsc) kernel, providing a strong security boundary between the untrusted code and the host OS.2.
2. The Communication LoopA single task execution follows this 5-step lifecycle:Instruction: Rails issues a POST /execute to the Node.js Orchestrator.Transport: Node.js attaches to the Docker container's stdin and stdout.Execution: The command (e.g., Python script) runs inside the gVisor sandbox.Multiplexing: Raw binary data (8-byte headers) is streamed back to Node.js and then to Rails.Reconciliation: Rails parses the stream (offsetting binary headers), updates the Workload database, and returns the final JSON to the user.
3. Security Model (The gVisor Layer)Unlike standard Docker containers that share the host Linux kernel, our agents use runsc.System Call Interception: gVisor intercepts syscalls from the Python/Alpine environment.Kernel Isolation: Even a "root" user inside the container cannot access host memory or the host file system.Network Fencing: Agents are restricted to a private bridge, preventing unauthorized "phone-home" DNS requests.
4. Database Schema (Persistence)We use a Workloads table to maintain a real-time "World Map" of the cluster.FieldTypeDescriptionagent_idStringUnique identifier for the container.statusStringEnum: running, terminated, error.internal_ipStringThe bridge IP for inter-agent communication.last_health_checkDateTimeTimestamp of the last successful heartbeat.
5. Key TechnologiesRuby on Rails: Orchestration Logic & State.Node.js (TypeScript): Low-latency Docker Socket Management.Docker Engine: Container Lifecycle.Google gVisor: User-space Kernel for Sandbox Security.PostgreSQL: Relational State Tracking.
